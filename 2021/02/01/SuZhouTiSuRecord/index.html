出于一些原因，关注苏州、AI在医疗领域的应用。

技术考查

  • 项目：对项目刨根问底，角度多是从数据角度，比如缓解某个算法应用的冷启动问题、以及模型评估、数据闭环管理如何反馈，如何快速应对业务变化及迁移等。不难理解，精细领域的小型公司遇到的问题一般是数据量小且AI算法起步难，还有就是业务变更频繁。这种公司多为乙方。不同的甲方爸爸需求不同，乙方儿子需要尽可能适应业务变化，最好有一套比较通用的解决方案，对于不同的需求，能够以最小的代价做出最快的适应，并且能够很快评估、调整，毕竟对于他们，风险是很难接受的。
  • 算法功底：Bagging VS Boosting 、方差-偏差困境、ROC曲线(如何挑选最优阈值)、模型评价指标(分类：查准、查全、F1-score；回归：误差平方和等)

代码考查

  • 考查对Python基础的了解，凭对python代码的理解，不要运行直观给出答案

    这个题考查的有两个点：

    1. Python 函数形参和实参的值传递和址传递，网上一般说法是“不可变对象”和“可变对象”，其实我觉得不太形象，如果有C或者C++基础，个人觉得“值传递和址传递”更透彻一些。参考资源；
    2. Python List中，append和“+”(extend)方法的差别，这一点小伙伴想往祖坟上刨，建议去看Python底层实现Cpython，即C语言实现的Python，其中笔者Python版本是3.7.4，下载源码，cpython-3.7.4/Objects/listobject.c，这个文件就是C语言实现的Python List底层源码，可以深刻扒扒append、extend 函数底层实现

    解读：

    1. Python函数中，形参如果是List，当实参List传入时，传入的是List的地址，函数操作时，改变List即使不返回任何，传入的List也会发生改变；形参如果是具体的数据原型，如int、float等，传入的是实参的值，即使形参名实参名相同，也仅仅是参数名相同，在内存中分配的地址不同，当你在函数内对着传入的实参一顿操作，其实操作的内存单元和存储真正实参的内存单元不同，所以当你不返回时，函数内部变量地址被释放，外边的传入实参不会发生改变，解释补充1-2、补充1-3。
    2. Python 中 List，append函数是在原来List扩充元素，返回的List和原来List的地址相同，只是对应C语言中数组大小变大了；而extend函数，即“+”是开辟新的存储地址，将两个list拼接之后，返回新地址指向的List。解释补充1-0、补充1-1。

    解读：

    1. 列表 * 操作，重复，但包含的子List，内存是一样的
    2. 中括号包围着的开辟新的List地址

    解读：参考资料

  • 考查numpy api熟悉程度 以及 简单实现 & 动态规划思路实现，numpy api 可以用来构建卷积神经网络

    请仅仅使用Python 和Numpy API，来写一个函数。这个函数用来计算一个一位数组的移动平均数（moving average）。也就是：
    输入：一维的python list叫做 X，由浮点数组成。同时输入一个整数k。
    输出：一维的python list 叫做Y。Y的数组索引为i，对于i>=k-1, Y[i] = average(X[i-k+1], X[i-k+2], …, X[i]) ，对于i<k-1, Y[i] = None。

    要求程序运行速度尽可能的快。

    例子：
    输入：k = 3, X = [-1，0，1，2，3]
    输出：Y = [None, None, 0, 1, 2]

    输入: k = 4, X = [-1, 0, 1, 2, 5]
    输出: Y = [None, None, None, 0.5, 2]

总结

代码测试，python项目中很多浅层bug由复制粘贴导致，很多深层bug是值传递、址传递、深拷贝、浅拷贝导致。对移动平均数的numpy api考查，抛开深度学习框架，如果有必要，用numpy自己构建卷积神经网络；对VGG网络的考查，充分理解卷积神经网络的作用机理，计算参数量和计算复杂度，体会卷积核作为特征提取器，共享参数的作用，以及比起全连接层，计算量更小的优势。
